# LangChain 白话总览（核心概念一把抓）

用最直白的说法把 **模型 I/O、RAG、存储、工具、回调** 串起来；细节与版本差异见 [核心架构与模块详解](./langchain_core_architecture_modules.md)。

---

## 1. 最先要搞懂的：模型 I/O

- **不是**「跟终端用户交互的 I/O」。
- **是**「专门跟大模型说话的**入口和出口**」。
- 典型链路：**拼提示词 → 发给大模型 → 把返回解析成程序里好用的结构**（字符串、JSON、列表等）。

---

## 2. RAG 是什么

- 可以理解为：**给大模型外挂一层「搜索 / 知识库」**。
- 流程（概念上）：
  **用户提问 → RAG 去检索资料 → 把检索结果拼进提示词 → 再通过模型 I/O 发给大模型生成回答**。
- 主要缓解：**不懂你的私有数据、知识过时、胡编（幻觉）** 等问题。

---

## 3. 另外三个模块（大白话）

### ① 存储模块（记忆）

- 存的是 **对话历史、上下文** 这类「会话状态」。
- **不是**向量库本体：FAISS、Chroma 等常用于 **RAG 的向量检索**；和「记会话」可以是不同一层。
- 存什么（示意）：用户 ID、会话 ID、角色（user/assistant）、消息内容、时间戳等。
- 用什么库：**普通数据库即可**——MySQL、MongoDB、Redis 等，按表或文档设计，没有「必须某一种」。

### ② 工具 / 技能（Tools）

- 就是 **给 AI 调用的函数接口**（封装好的业务能力）。
- 需要查天气、算数、查库、调 API 时，由模型在流程里 **选择并调用** 这些接口。
- 理解成 **「可被 AI 调用的业务函数」** 就够用。

### ③ 回调（Callbacks）

- 和 **Vue 3 生命周期** 的类比（便于记忆）：
  - `onMounted` / `onUpdated` / `onUnmounted`
  - 对应 LangChain 里类似 **`on_chain_start`、`on_llm_start`、`on_tool_end`、`on_chain_end`** 等（具体名称以文档为准）。

- **不是**按「用户聊了几轮」或「时间」触发；
- **是**按 **LangChain 内部执行到哪一个步骤** 触发：链开始 → 检索 → 调工具 → 调大模型 → 生成 token → 结束，**每一步**都可以挂钩子。

一次问答里，**依次**可能触发（示例顺序，非固定）：

1. `on_chain_start`（整条链开始）  
2. `on_retriever_start` / `on_retriever_end`（RAG 检索开始与结束）  
3. `on_llm_start`（大模型开始生成）  
4. `on_llm_new_token`（流式时每个 token 都可能触发）  
5. `on_chain_end`（流程结束）  

在任意一步插你自己的逻辑：**日志、监控、报错、推流** 等。

---

## 4. 一句话超级总结

| 概念 | 一句话 |
|------|--------|
| **I/O** | 跟大模型对话的口子（进提示、出解析） |
| **RAG** | 先帮大模型查资料，再生成 |
| **存储** | 记对话历史与会话状态 |
| **工具** | AI 能调用的业务函数 |
| **回调** | 流程每一步的「钩子」，类似生命周期，可插日志/监控/流式 |
